============================================================
一、项目目标
============================================================

本项目为高校青年志愿者协会设计，旨在自动化处理德育分加分数据的录入与名单生成流程。具体目标：

1. 将 Word 格式的加分名单（由活动负责人提交）自动解析并录入到 Excel 年级花名册中；
2. 从 Excel 花名册中读取指定活动的加分数据，按排版规则生成格式化的 Word 加分名单文档；
3. 提供图形化操作界面，使不熟悉命令行的人员也能轻松使用；
4. 支持多次活动导入，数据在同一份 Excel 中累积管理。

============================================================
二、项目实现方法
============================================================

2.1 技术选型
- 语言：Python 3
- Excel 处理：openpyxl
- Word 处理：python-docx + OOXML 底层操作
- 图形界面：CustomTkinter（基于 Tkinter 的现代化封装）
- 启动方式：Windows .bat 批处理文件，双击即可运行

2.2 核心模块

【导入模块 import_scores.py】
- 解析 Word 加分名单，识别分数标题、班级名、学生姓名
- 姓名匹配采用正则宽松匹配（\s* 插值），兼容两字姓名中间的空格变体
- 按班级写入 Excel 对应 Sheet 的日期列，支持多次导入追加新列
- 同日期重复导入时自动覆盖旧数据

【生成模块 generate_list.py】
- 从 Excel 读取指定日期的加分数据
- 按规则排序：分数降序 → 年级优先 → 专业优先级 → 姓名长度
- 生成 Word 文档，实现精确排版：
  · 中文 → 宋体，数字/括号/点 → Times New Roman（逐字符自动分类）
  · 悬挂缩进 6.25 字符（纯 twips 定位，避免 OOXML 字符单位舍入偏差）
  · <w:noWrap/> 防止姓名跨行断裂
  · 分数标题加粗且不缩进

【工具模块 utils.py】
- 班级排序键：年级 23 > 24 > 25，同级专业 地卓 > 地师 > 地理 > 地信 > 环境
- 姓名长度排序键：3 字 > 2 字 > 4 字 > 5 字 > 6 字 > 含点(·)
- 中文字符判断（CJK 统一表意文字范围）

【图形界面 gui.py】
- 主界面：生成名单区域（Excel 文件选择 + 活动日期输入 + 一键生成）
- 工具栏下拉菜单：
  · 导入分数 → 弹窗（Word 文件选择 + Excel 目标选择）
  · 生成名单-测试 → 弹窗（风险提示 + 自由选择 Excel 源）
- 快捷操作：最近文件、打开输出文件夹
- 子进程通过 cmd /k 在新终端窗口中运行，结果可见

2.3 关键难点及解决方案

| 难点                           | 解决方案                                                      |
|--------------------------------|-------------------------------------------------------------|
| Word 续行悬挂缩进首行偏移       | 仅使用 twips 值（w:left/w:firstLine），删除字符单位属性       |
| 四字姓名在行末被拆开            | 对姓名 run 添加 <w:noWrap/> 禁止换行                         |
| 中英文数字混排字体              | 按字符类型分组，逐 run 设置对应字体                           |
| 两字姓名中间空格与其他空格混淆  | 两字姓名中间固定 2 个空格，空格独立成组不跟随相邻字符         |
| 排序与模板不一致                | 年级优先 → 专业优先级 → 姓名长度，与模板完全对齐              |
| GUI 子进程启动 python 失败      | sys.executable 路径不可靠，改为列表传参 ['cmd','/k','python',...] |
| 多次导入数据丢失                | 导入时优先读取 _updated.xlsx，后续导入在已有数据上追加新列    |

三、项目成果
============================================================

3.1 交付物

├── gui.py                    # 图形界面主程序
├── import_scores.py          # Word → Excel 导入脚本
├── generate_list.py          # Excel → Word 生成脚本
├── utils.py                  # 共享工具模块
├── launcher.py               # 命令行交互式菜单（备用）
├── 启动.bat                  # 双击启动器
├── 年级花名册数据表.xlsx      # 学生数据模板
├── 文件模板/                 # Word 名单模板存放目录
└── 输出文件夹/               # 生成结果输出目录

3.2 功能清单

- 导入分数：选择 Word 文件 → 自动解析 → 写入 Excel 花名册
- 生成名单：选择 Excel 文件 → 输入日期 → 生成格式化 Word 名单
- 多次导入：支持多个活动分数在同一 Excel 中累积（不同日期列）
- 自动覆盖：同日期重复导入自动覆盖旧数据
- 测试模式：工具栏提供"生成名单-测试"，可自由选择任意 Excel 源文件
- 一键预览：工具栏提供"最近文件"和"打开输出文件夹"快捷操作
- 双重入口：图形界面（推荐）+ 命令行菜单（备用）

3.3 排版效果
- 生成 Word 文档的字体、缩进、换行行为与手动排版的模板名单一致
- 分数标题加粗顶格，班级行悬挂缩进 6.25 字符
- 中文字符用宋体，数字和括号用 Times New Roman

============================================================
四、项目意义
============================================================

4.1 效率提升
传统手动处理一次活动加分名单约需 30-60 分钟（打开 Excel → 逐班查找学生 → 录入分数 → 手动排版 Word），使用本工具后缩短至 1 分钟以内。

4.2 降低门槛
此前操作需要同时熟悉 Excel、Word 排版规则和 Python 命令行，现通过图形界面实现"选文件 → 点按钮"的极简操作，任何部员均可使用。

4.3 降低出错率
- 姓名匹配采用字典 + 正则双重保障，减少手动录入的遗漏和错别字
- 排版参数程序化控制，消除手动调整的不一致性
- 数据集中管理于一份 Excel，避免多份文件版本混乱

4.4 可扩展性
- 模块化设计：导入/生成/工具各模块独立，便于单独维护
- Sheet 驱动：新增班级只需在 Excel 模板中添加 Sheet
- 日期驱动：新增活动只需导入对应的 Word 文件，自动创建日期列
